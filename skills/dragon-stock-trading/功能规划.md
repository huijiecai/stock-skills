## 按日数据采集流程设计

### Step 1：拉取指定日期涨停个股

每日脚本需要：

1. 拉取今日涨停股票列表

2. 对每只股票：

   * 若 stock_info 不存在 → 补充基础信息
   * 写入 stock_daily（行情数据）
   * 更新或生成 stock_events（如涨停、炸板等事件）

3. 计算并更新 market_sentiment：

   * 涨停数量
   * 连板数量
   * 跌停数量
   * 炸板率
   * 情绪周期判断（可后续扩展）

## 二、同花顺概念数据采集（重构）

> **核心目标**：识别情绪拐点时资金认可的方向 + 主动性最好的个股

### 数据来源

使用同花顺概念接口（需 6000 积分）：

| 接口 | 功能 | 用途 |
|------|------|------|
| `ths_index` | 概念指数列表 | 获取所有概念名称、代码、成分数量 |
| `ths_member` | 概念成分股 | 获取某概念包含哪些股票 |
| `ths_daily` | 概念日行情 | 获取概念涨跌幅、成交额 |

### 新增数据表

#### 1. ths_concept - 同花顺概念列表

> 数据来源：`pro.ths_index()` - [文档](https://tushare.pro/document/2?doc_id=259)

```sql
CREATE TABLE ths_concept (
    ts_code TEXT PRIMARY KEY,      -- 概念代码（如 885835.TI）
    name TEXT NOT NULL,            -- 概念名称
    concept_type TEXT,             -- 类型：N=概念/I=行业/R=地域
    component_count INTEGER,       -- 成分股数量
    list_date TEXT,                -- 发布日期
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

#### 2. ths_concept_member - 概念成分股

> 数据来源：`pro.ths_member()` - [文档](https://tushare.pro/document/2?doc_id=261)

```sql
CREATE TABLE ths_concept_member (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    concept_code TEXT NOT NULL,    -- 概念代码
    concept_name TEXT NOT NULL,    -- 概念名称
    stock_code TEXT NOT NULL,      -- 股票代码
    stock_name TEXT,               -- 股票名称
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(concept_code, stock_code)
);

CREATE INDEX idx_member_concept ON ths_concept_member(concept_code);
CREATE INDEX idx_member_stock ON ths_concept_member(stock_code);
```

#### 3. ths_concept_daily - 概念日行情

> 数据来源：`pro.ths_daily()` - [文档](https://tushare.pro/document/2?doc_id=260)

```sql
CREATE TABLE ths_concept_daily (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    trade_date TEXT NOT NULL,      -- 交易日期

    pre_close REAL,                -- 昨日收盘点位
    open REAL,                     -- 开盘点位

    high REAL,                     -- 最高点位

    pct_change REAL,               -- 涨跌幅

    turnover_rate REAL,            -- 换手率

    float_mv REAL,                 -- 流通市值（亿元）
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(trade_date, concept_code)

CREATE INDEX idx_concept_daily_date ON ths_concept_daily(trade_date);
```

#### 4. ths_hot_rank - 个股热榜

> 数据来源：`pro.ths_hot()` - [文档](https://tushare.pro/document/2?doc_id=320)
> 
> **特点**：每日盘中提取 4 次，收盘后 4 次，最晚 22 点更新

**核心价值**：龙头战法的"人气底线"判断依据

**字段说明**：

| 字段 | 说明 | 用途 |
|------|------|------|
| `hot` | 热度值 | 核心指标！量化人气 |
| `rank` | 排名 | 判断是否进入前 30 |
| `concept` | 概念标签 | 快速知道炒作逻辑 |
| `rank_reason` | 上榜解读 | 理解为什么热 |
| `pct_change` | 涨跌幅 | 结合热度判断 |

```sql
CREATE TABLE ths_hot_rank (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    trade_date TEXT NOT NULL,      -- 交易日期
    rank_time TEXT NOT NULL,       -- 排行榜时间（HH:MM:SS）
    ts_code TEXT NOT NULL,         -- 股票代码
    ts_name TEXT,                  -- 股票名称
    rank INTEGER,                  -- 排名
    hot REAL,                      -- 热度值
    pct_change REAL,               -- 涨跌幅
    current_price REAL,            -- 当前价格
    concept TEXT,                  -- 概念标签（JSON 数组）
    rank_reason TEXT,              -- 上榜解读
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    UNIQUE(trade_date, rank_time, ts_code)
);

CREATE INDEX idx_ths_hot_rank_date ON ths_hot_rank(trade_date);
CREATE INDEX idx_ths_hot_rank_stock ON ths_hot_rank(ts_code);
```

#### 5. ths_limit_list - 涨跌停榜单（连板天梯）

> 数据来源：`pro.limit_list_ths()` - [文档](https://tushare.pro/document/2?doc_id=355)
> 
> **特点**：历史数据从 2023年11月开始，每天 16 点更新

**核心价值**：龙头候选池 + 连板天梯

**关键字段**：

| 字段 | 说明 | 用途 |
|------|------|------|
| `status` | 涨停状态 | N连板、一字板 |
| `lu_desc` | 涨停原因 | 理解炒作逻辑 |
| `tag` | 涨停标签 | 首板/2板/3板 |
| `limit_type` | 板单类别 | 涨停池/连扳池/炸板池/跌停池 |
| `limit_order` | 封单量 | 资金认可度 |
| `first_lu_time` | 首次涨停时间 | 判断主动性 |

```sql
CREATE TABLE ths_limit_list (
    id INTEGER PRIMARY KEY AUTOINCREMENT,
    trade_date TEXT NOT NULL,      -- 交易日期
    ts_code TEXT NOT NULL,         -- 股票代码
    ts_name TEXT,                  -- 股票名称
    price REAL,                    -- 收盘价
    pct_chg REAL,                  -- 涨跌幅
    limit_type TEXT NOT NULL,      -- 板单类别：涨停池/连扳池/炸板池/跌停池
    tag TEXT,                      -- 涨停标签：首板/2板/3板
    status TEXT,                   -- 涨停状态：N连板、一字板
    lu_desc TEXT,                  -- 涨停原因
    open_num INTEGER,              -- 打开次数
    first_lu_time TEXT,            -- 首次涨停时间
    last_lu_time TEXT,             -- 最后涨停时间
    limit_order REAL,              -- 封单量（股）
    limit_amount REAL,             -- 封单额（元）
    lu_limit_order REAL,           -- 最大封单（股）
    turnover_rate REAL,            -- 换手率
    turnover REAL,                 -- 成交额
    free_float REAL,               -- 实际流通市值
    sum_float REAL,                -- 总市值
    limit_up_suc_rate REAL,        -- 近一年涨停封板率
    market_type TEXT,              -- 股票类型：HS/GEM/STAR

    UNIQUE(trade_date, ts_code, limit_type)

CREATE INDEX idx_ths_limit_date ON ths_limit_list(trade_date);
CREATE INDEX idx_ths_limit_stock ON ths_limit_list(ts_code);
CREATE INDEX idx_ths_limit_type ON ths_limit_list(limit_type);
```

### 采集脚本设计

```bash
# 使用统一的采集脚本（与 collect_stock_data.py 风格一致）

# 1. 拉取概念列表
python collect_ths_data.py --method concept_list

# 2. 拉取概念成分股
python collect_ths_data.py --method members

# 3. 拉取概念日行情
python collect_ths_data.py --method daily --start 2026-03-01 --end 2026-03-02

# 4. 拉取个股热榜（历史数据）
python collect_ths_data.py --method hot_rank --start 2026-03-01 --end 2026-03-02

# 5. 拉取个股热榜（最新，22:30 更新）
python collect_ths_data.py --method hot_rank --is-new Y

# 6. 拉取涨跌停榜单（连板天梯）
python collect_ths_data.py --method limit_list --start 2026-03-01 --end 2026-03-02

# 7. 拉取连板池（只采集连板股）
python collect_ths_data.py --method limit_list --limit-type 连扳池

# 8. 采集全部概念数据
python collect_ths_data.py --method all --days 60
```

**命令行参数说明**：

| 参数 | 说明 |
|------|------|
| `--method` | 采集方法：concept_list=概念列表, members=成分股, daily=日行情, hot_rank=个股热榜, limit_list=涨跌停榜单, all=全部 |
| `--start` | 开始日期（YYYY-MM-DD） |
| `--end` | 结束日期（YYYY-MM-DD） |
| `--days` | 采集最近 N 天的数据（默认 60 天） |
| `--is-new` | 热榜专用：Y=最新数据（22:30更新），N=盘中数据 |
| `--limit-type` | 涨停池/连扳池/炸板池/跌停池，默认：涨停池 |
| `--force` | 强制重新采集 |

def find_money_attack_direction(date: str, time: str = None) -> List[Dict]:

    1. 板块异动监控（分时）：

       - 分时站稳均线（承接好）

       - 板块内涨停家数

## 四、concept_daily 生成逻辑（调整）

### 数据来源变更

**原方案**：基于 `stock_concept` 表计算

**新方案**：直接使用 `ths_concept_daily`（同花顺官方数据）

### 增强计算逻辑

基于 `ths_concept_daily` + `stock_daily` 计算：

1. 板块强度 = 涨停数 × 连板高度 × 成交额
2. 领涨股识别 = 涨幅最大 + 涨停时间最早
3. 板块联动度 = 成分股上涨比例
