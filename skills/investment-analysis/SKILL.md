# 投资分析 Skill

> 为 AI 提供系统化的公司分析、行业研究和概念池管理能力

---

## 功能定义

本 skill 赋予 AI 三项核心能力：

### 1. 公司分析能力
生成标准化的个股分析报告，包括主营业务、财务数据、行业地位、投资亮点和风险提示。

**触发条件**：
- 用户提到公司名称或股票代码
- 用户明确要求分析某公司
- 用户询问某公司情况

**执行要求**：
- 使用 `scripts/` 获取实时数据（优先）
- 使用 Web 搜索补充定性信息
- 使用 `templates/company-template.md` 生成报告
- 保存到 `docs/[行业]/[公司名称].md`

**参考**：[操作指南](reference/1-公司分析/操作指南.md)

---

### 2. 行业分析能力
生成产业链/细分行业研究报告，覆盖市场规模、竞争格局、A股投资标的全景。

**触发条件**：
- 用户提到行业名称
- 用户询问产业链
- 用户讨论技术趋势

**执行要求**：
- 收集行业信息（市场规模、竞争格局、上市公司）
- 使用 `templates/industry-template.md` 生成行业报告
- 保存到 `docs/[行业]/00_行业分析.md`
- **自动触发**：批量分析行业内公司（参考批量分析指南）
- **自动触发**：更新概念池

**参考**：[操作指南](reference/2-行业分析/操作指南.md) / [批量分析指南](reference/2-行业分析/批量分析指南.md)

---

### 3. 概念池管理能力
基于分析结果，将公司归入对应概念池，维护 `docs/概念股票池体系.md`。

**触发条件**：
- 完成公司/行业分析后**主动询问**用户是否更新
- 用户明确要求更新概念池

**执行要求**：
- 判定核心/相关标的（参考判定标准）
- **必须**经过用户确认后才更新
- 记录归属逻辑和信息来源
- 更新 `docs/概念股票池体系.md`

**参考**：[更新指南](reference/3-概念池管理/更新指南.md) / [归属判定](reference/3-概念池管理/归属判定.md)

---

## 执行规范

### 数据获取顺序
1. **优先**：使用 `scripts/realtime_data.py` 获取实时数据（秒级响应）
2. **补充**：使用 Web 搜索获取定性信息（战略、竞争力等）
3. **验证**：交叉验证多个数据源

详见：[公司分析操作指南](reference/1-公司分析/操作指南.md) / [行业分析操作指南](reference/2-行业分析/操作指南.md)

### 文件组织规范
- 公司报告：`docs/[行业]/[细分领域]/[公司名称].md`
- 行业报告：`docs/[行业]/00_行业分析.md`
- 概念池：`docs/概念股票池体系.md`

详见：[目录规范](reference/规范与标准/目录规范.md)

### 质量标准
- **数据准确**：所有数据标注来源和时间
- **分析完整**：必须包含财务对比、投资逻辑、风险提示
- **格式规范**：使用标准模板，保持一致性
- **客观中立**：避免主观臆断，平衡优缺点

详见：[质量标准](reference/规范与标准/质量标准.md)

## 重要约束

1. **免责声明**：所有分析仅供学习研究，不构成投资建议
2. **风险提示**：每份报告必须包含风险分析，不能只谈优点
3. **用户确认**：概念池更新必须经过用户明确确认
4. **数据时效**：注明数据获取时间，避免使用过时数据
5. **客观中立**：保持分析客观性，避免情绪化表达
