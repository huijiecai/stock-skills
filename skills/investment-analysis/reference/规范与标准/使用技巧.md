## 深度 vs 快速分析

### 深度分析（推荐）

**适用场景**：
- 用户明确要求"详细分析"、"深度研究"
- 重要投资决策参考
- 首次分析某公司/行业

**特点**：
- 完整的信息收集（30-60分钟）
- 详细的财务分析（至少3年数据）
- 全面的风险评估
- 使用完整模板

**工作流**：
1. 实时数据获取（1-2分钟）
2. 深度Web搜索（20-30分钟）
   - 官方公告、研报、行业报告
   - 多个信息源交叉验证
3. 详细报告生成（10-15分钟）
4. 质量检查和完善

### 快速分析（可选）

**适用场景**：
- 用户明确要求"快速了解"、"简单看看"
- 初步筛选阶段
- 已有基础了解，仅需更新数据

**特点**：
- 仅包含核心信息（10-15分钟）
- 主营业务、财务摘要、估值
- 标注为"快速分析版"

**工作流**：
1. 实时数据获取（1-2分钟）
2. 简要Web搜索（5-8分钟）
   - 主要看官网和最新公告
3. 核心报告生成（5分钟）
4. 标注"快速分析版，可后续扩展"

**示例标注**：
```markdown
# 科大讯飞 - 快速分析版

> ⚠️ 本报告为快速分析版，仅包含核心信息
> 如需详细分析，请使用"深度分析"功能

## 核心信息

**主营业务**：AI语音识别、大模型应用
**最新财务**：营收XXX亿，净利润XXX亿（2025Q3）
**估值水平**：PE(TTM) XX，PB XX
**投资亮点**：AI语音龙头，讯飞星火大模型
**主要风险**：商业化进度、竞争加剧

详细分析待补充...
```

### 选择建议

- **默认使用深度分析**，确保信息完整准确
- 仅在用户明确要求时使用快速分析
- 快速分析后可随时升级为深度分析

## 持续更新

### 触发更新的场景

1. **财报发布**
   - 年报/季报发布后
   - 更新财务数据和业绩分析
   - 调整业绩预期

2. **重大事件**
   - 并购重组
   - 产品/技术突破
   - 政策重大变化
   - 管理层变动

3. **定期维护**
   - 每季度检查数据时效性
   - 更新行业竞争格局
   - 调整投资评级

### 更新流程

**步骤1：检查需要更新的内容**
```markdown
检查清单：
- [ ] 财务数据过期（>3个月）
- [ ] 估值数据过期（>1周）
- [ ] 重大事件未记录
- [ ] 行业地位变化
- [ ] 风险提示需调整
```

**步骤2：更新内容**
- 获取最新数据（实时数据 + Web搜索）
- 更新对应章节
- 保持其他章节不变

**步骤3：标注更新信息**
```markdown
# 科大讯飞

> 更新日期：2026-03-15
> 上次更新：2025-12-20
> 更新内容：2025年报数据、业绩预期调整

---
```

### 更新频率建议

| 内容类型 | 更新频率 | 触发条件 |
|---------|---------|---------|
| 财务数据 | 季度 | 财报发布后1周内 |
| 估值数据 | 月度 | 股价大幅波动（±20%） |
| 行业地位 | 半年度 | 行业报告发布 |
| 投资逻辑 | 年度 | 重大变化时随时 |
| 风险提示 | 年度 | 新风险出现时随时 |

## 数据获取优化

### 优先级策略

**第一优先级：实时数据API**
```python
# 快速获取基础数据（1-2分钟）
from scripts.realtime_data import get_stock_realtime_data, get_stock_financial_data

realtime = get_stock_realtime_data('600519')
financial = get_stock_financial_data('600519')
```

**优势**：
- 秒级响应
- 数据准确
- 自动计算指标

**第二优先级：Web搜索补充**
- 定性分析（战略、竞争力）
- 最新动态（公告、新闻）
- 深度研究（研报、行业报告）

### 缓存机制

**适用场景**：
- 短时间内重复查询同一股票
- 批量分析时避免重复请求

**实现示例**：
```python
import json
from pathlib import Path
from datetime import datetime, timedelta

CACHE_DIR = Path('cache')
CACHE_DURATION = timedelta(hours=1)

def get_with_cache(code):
    """带缓存的数据获取"""
    cache_file = CACHE_DIR / f"{code}.json"
    
    # 检查缓存
    if cache_file.exists():
        with open(cache_file) as f:
            cached = json.load(f)
            cache_time = datetime.fromisoformat(cached['timestamp'])
            if datetime.now() - cache_time < CACHE_DURATION:
                return cached['data']
    
    # 获取新数据
    data = get_stock_realtime_data(code)
    
    # 保存缓存
    if data:
        CACHE_DIR.mkdir(exist_ok=True)
        with open(cache_file, 'w') as f:
            json.dump({
                'timestamp': datetime.now().isoformat(),
                'data': data
            }, f)
    
    return data
```

**注意事项**：
- 缓存时长不宜过长（建议1小时）
- 财报日当天避免使用缓存
- 定期清理过期缓存

## 多公司对比

### 场景：同行业公司估值对比

**示例**：白酒行业龙头对比

```python
codes = ['600519', '000858', '000568']  # 茅台、五粮液、泸州老窖

results = []
for code in codes:
    data = get_stock_realtime_data(code)
    if data:
        results.append({
            'name': data['name'],
            'code': code,
            'pe_ttm': data.get('pe_ttm', 'N/A'),
            'pb': data.get('pb', 'N/A'),
            'total_mv': data.get('total_mv', 0) / 10000,
        })

# 生成对比表格
print(f"{'公司':<10} {'代码':<10} {'PE(TTM)':<10} {'PB':<10} {'市值(亿)':<12}")
for r in results:
    print(f"{r['name']:<10} {r['code']:<10} {r['pe_ttm']:<10} {r['pb']:<10} {r['total_mv']:<12.2f}")
```

**输出**：
```
公司         代码         PE(TTM)    PB         市值(亿)       
贵州茅台      600519     19.66      7.80       17702.59    
五粮液       000858     14.05      3.02       3996.12     
泸州老窖      000568     12.56      3.33       1587.64     
```

**分析要点**：
- 估值对比：茅台PE最高，反映市场认可度
- 市值对比：茅台遥遥领先，体现龙头地位
- 投资建议：结合估值和成长性选择

## 常见问题处理

### 问题1: 找不到公司资料

**解决方案**：
1. 确认股票代码正确
2. 检查是否为新股（上市<1年）
3. 尝试搜索公司全称
4. 查看是否已退市或暂停上市

### 问题2: 财务数据差异大

**解决方案**：
1. 确认统计口径（合并报表 vs 母公司）
2. 检查报告期是否一致
3. 使用官方公告数据为准
4. 标注数据差异原因

### 问题3: 估值难以判断

**解决方案**：
1. 对比同行业公司
2. 查看历史估值中枢
3. 考虑行业特性（周期性、成长性）
4. 不强制给出买入/卖出建议

## 最佳实践总结

1. **优先实时数据，Web搜索补充**
2. **深度分析为主，快速分析为辅**
3. **定期更新，保持数据时效性**
4. **使用缓存，避免重复请求**
5. **批量操作，提高效率**
6. **质量检查，确保准确性**
7. **文档规范，便于维护**
