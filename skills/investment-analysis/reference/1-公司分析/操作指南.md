# 公司分析操作指南

> 详细说明如何分析单个公司并生成分析报告

---

## 步骤1: 信息收集（30-40分钟）

### 优先使用实时数据获取

```python
from scripts.realtime_data import get_stock_realtime_data, get_stock_financial_data
from scripts.stock_formatter import format_for_analysis

# 1. 获取实时行情和估值数据
realtime = get_stock_realtime_data('600519')  # 6位股票代码
# 包含：基本信息、最新行情、PE/PB/PS、市值等

# 2. 获取财务数据
financial = get_stock_financial_data('600519')
# 包含：营收、利润、资产负债、现金流等

# 3. 格式化为 Markdown（可选）
markdown = format_for_analysis(realtime, financial)
```

**优势**：
- 秒级响应，无需等待网络搜索
- 数据准确，来自官方API
- 自动计算估值指标和财务比率
- 避免搜索引擎的缓存延迟

**配置要求**：需要配置 Tushare Token

### 补充使用Web搜索（用于深度信息）

**官方信息**（最高权重）：
- 公司官网：主营业务、产品线、技术实力
- 巨潮资讯网：最新年报、季报、公告
- 交易所网站：重大事项、投资者关系

**搜索关键词**：
```
- "[公司名称] 官网"
- "[股票代码] 年报 2025"
- "[公司名称] 主营业务"
- "[公司名称] 研报 2025"
- "[公司名称] 技术突破 最新进展"
- "[公司名称] 核心竞争力"
- "[公司名称] 行业地位"
- "[公司名称] 市占率 排名"
```

**专业研究**（高权重）：
- 券商研报：行业地位分析、估值模型
- 行业报告：竞争格局、技术趋势
- 专业媒体：公司动态、重大事件

**市场讨论**（参考）：
- 雪球：投资者讨论、一手信息
- 财联社：实时快讯、行业动态
- 36氪/钛媒体：科技公司深度报道
- 华尔街见闻：宏观市场分析

**信息交叉验证**（必须）：
- ✅ 至少2个独立来源验证关键数据
- ✅ 官方公告 > 券商研报 > 媒体报道 > 市场讨论
- ✅ 财务数据必须以官方年报/季报为准
- ✅ 行业地位需多方信息印证
- ⚠️ 市场讨论仅作参考，不作为核心依据

**建议工作流**：
1. 先用 `scripts/realtime_data.py` 获取基础数据（1分钟）
2. 再用 Web 搜索补充深度信息（定性分析、战略规划等）
3. **交叉验证**：对比多个信息源，确保数据准确性
4. 结合两者生成完整报告

**验证重点**：
- 财务数据：对比实时API + 官方公告
- 市场地位：对比券商研报 + 行业报告
- 最新动态：对比财联社 + 公司公告
- 市场认知：参考雪球等社区讨论

## 步骤2: 生成报告（使用模板）

使用 `templates/company-template.md` 生成标准化报告，包含：

1. **公司简介**（表格形式）
   - 证券代码、上市板块
   - 成立时间、上市时间
   - 所属行业、市场地位

2. **主营业务**
   - 核心产品/服务
   - 技术路线
   - 业务模式

3. **财务与业绩**
   - 最新年报数据（营收、净利润、毛利率等）
   - 最新季报数据
   - 同比/环比变化分析

4. **行业地位与产业链定位**
   - 产业链位置
   - 市占率/排名
   - 竞争优势

5. **投资亮点**
   - 核心竞争力
   - 成长驱动因素
   - 政策/行业红利

6. **风险提示**
   - 业绩风险
   - 竞争风险
   - 估值风险

7. **业绩预期与估值**
   - 未来1-2年业绩预期
   - PE/PB估值水平
   - 关键观察指标

---

## 步骤3: 保存文档

保存到：`docs/[行业名称]/[细分领域]/[公司名称].md`（大型行业需要细分）
或：`docs/[行业名称]/[公司名称].md`（小型行业可直接放在行业目录下）

详细目录组织规范参见 [../规范与标准/目录规范.md](../规范与标准/目录规范.md)

## 时间估算

- 实时数据获取：1-2 分钟
- Web搜索补充：15-20 分钟
- 报告生成：10-15 分钟
- **总计：30-40 分钟**
