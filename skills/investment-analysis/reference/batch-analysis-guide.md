## 工作流程概览

### 默认策略：全景扫描 + 拐点捕捉型龙头战法

当用户提出"分析xx行业"时，**默认采用全景扫描策略**，而非仅分析头部龙头。

**策略原则**：
- 覆盖所有主要细分领域（每个细分至少1-2家代表）
- 包含细分冠军、拐点受益标的，不仅限于市值龙头
- 按细分领域组织，采用批次化并行分析
- 大型行业使用fast model加速处理

### 完整流程图

```
用户触发"分析xx行业"
    ↓
[阶段1] 行业分析（全景扫描）
    ├─ 收集行业信息
    ├─ 识别所有相关A股公司（按行业规模：15-150家）
    │   ├─ 小型行业：15-25家
    │   ├─ 中型行业：30-50家
    │   └─ 大型行业：100-150家 ⭐
    ├─ 按层级和细分领域分类
    └─ 生成行业报告（带占位符链接）
    ↓
[阶段2] 公司批量分析（自动触发，全景扫描）
    ├─ 小型行业：分2-4批，常规model
    ├─ 中型行业：分3-5批，常规model
    └─ 大型行业：分5-8批，fast model ⭐
    ↓
[阶段3] 链接更新
    ├─ 扫描生成的公司报告
    ├─ 更新行业报告中的链接
    └─ 添加分析完成标记
    ↓
[阶段4] 概念池更新（自动触发）
    ├─ 批量提取公司-概念关系
    ├─ 调用概念池更新API
    └─ 更新验证状态
    ↓
完成：行业报告 + 公司报告 + 概念池更新
```

### 时间估算

| 阶段 | 小型行业 | 中型行业 | 大型行业 | 说明 |
|------|----------|----------|----------|------|
| 行业分析 | 30-40分钟 | 40-60分钟 | 60-90分钟 | 信息收集、报告生成 |
| 公司批量分析 | 30-60分钟 | 60-120分钟 | 120-180分钟 | 并行分析，fast model加速 |
| 链接更新 | 2-5分钟 | 2-5分钟 | 5-10分钟 | 自动化处理 |
| 概念池更新 | 5-10分钟 | 5-10分钟 | 10-15分钟 | 批量添加股票 |
| **总计** | **1-2小时** | **2-3.5小时** | **3-5小时** | 全自动化执行 |

## 执行阶段详解

### 阶段1: 行业分析

#### 1.1 公司识别与分类（全景扫描策略）

**识别目标**：

根据行业规模，确定公司覆盖数量：

| 行业类型 | 目标公司数 | 典型行业 | 策略 |
|---------|-----------|---------|------|
| 小型行业 | 15-25家 | CPO、连接器、特种气体 | 全覆盖 |
| 中型行业 | 30-50家 | 半导体设备、光伏、动力电池 | 细分领域覆盖 |
| **大型行业** | **100-150家** | **化工、有色、机械、建材** | **细分领域全景扫描** ⭐ |

**识别标准**：
- 该公司主营业务与该行业直接相关
- 在A股上市（包括主板、创业板、科创板、北交所）
- 有明确的市占率或产能数据
- 被主流券商研报提及
- **包含细分冠军和拐点受益标的，不仅限于市值龙头**

**分层级分类**：

| 层级 | 小型行业 | 中型行业 | 大型行业 | 入选标准 |
|------|----------|----------|----------|----------|
| 第一梯队 | 3-5家 | 5-8家 | 10-15家 | 市占率前三 / 营收前五 / 技术领先 |
| 第二梯队 | 8-12家 | 15-25家 | 50-80家 | 细分龙头 / 区域龙头 / 快速成长 |
| 第三梯队 | 4-8家 | 10-17家 | 40-55家 | 新进入者 / 转型升级 / 技术突破 |
| **总计** | **15-25家** | **30-50家** | **100-150家** | **全景扫描覆盖** |

**按细分领域分类**：

根据产业链环节或技术路线，将公司归入不同细分领域：
- 上游（资源开采）
- 中游（冶炼/制造）
- 下游（深加工/应用）

#### 1.2 生成行业报告

使用 `templates/industry-template.md` 生成标准化报告，特别注意：

- **第6章"A股投资标的全景"**：
  - 6.1 分层级分类（三个梯队）
  - 6.2 按细分领域分类
  - 6.3 公司综合对比矩阵
  - 6.4 投资标的选择建议

- **链接占位符**：
  - 在"详细分析"列使用占位符：`[📄 查看详情](../公司分析/[公司名称].md)`
  - 后续批量分析完成后自动更新

### 阶段2: 公司批量分析

#### 2.1 分批策略（按行业规模）

**分批原则**：
- 根据行业规模选择分批策略
- 大型行业按细分领域分批，而非按层级分批
- 每批数量合理，避免资源过载
- 最多同时运行5个并行任务

**小型行业分批示例**（15-25家，如CPO行业）：

#### 2.2 并行任务执行

**根据行业规模选择策略**：

| 行业类型 | 并行任务数 | Model选择 | 每批公司数 | 预计时间 |
|---------|-----------|----------|-----------|---------|
| 小型行业 | 2-3个 | 常规 | 3-5家 | 30-60分钟 |
| 中型行业 | 3-4个 | 常规 | 5-10家 | 60-120分钟 |
| **大型行业** | **5个** | **fast** | **20-25家** | **120-180分钟** |

**使用 Task tool 创建并行任务**：

**小型/中型行业**（常规model）：
```python
# 伪代码示例 - 常规行业
for batch in [batch1, batch2, batch3]:
    tasks = []
    for company_group in batch:
        task = Task(
            description=f"分析公司: {company_group}",
            prompt=f"""
            请使用 investment-analysis skill 的"一、公司分析"流程，
            为以下公司生成详细分析报告：
            
            公司列表：{company_group}
            
            要求：
            1. 使用 templates/company-template.md 模板
            2. 保存到 docs/[行业名称]/[细分领域]/[公司名称].md
            3. 包含完整的7个章节
            4. 确保数据准确性和来源可靠性
            5. 在"业绩预期与估值"章节标注拐点逻辑
            """,
            readonly=False,
            subagent_type="generalPurpose"
            # 不指定model，使用常规model
        )
        tasks.append(task)
    
    # 并行执行当前批次
    run_parallel(tasks)
```

**大型行业**（fast model）⭐：
```python
# 伪代码示例 - 大型行业（化工、机械等）
batches = {
    "农药": ["扬农化工", "利尔化学", ...],  # 20家
    "化纤": ["桐昆股份", "恒逸石化", ...],  # 15家
    "化肥+涂料+橡胶": ["盐湖股份", "新洋丰", "三棵树", ...],  # 18家
    "塑料+精细化工": ["金发科技", "万润股份", ...],  # 24家
    "无机化工+其他": ["三友化工", "中盐化工", ...],  # 25家
}

tasks = []
for field_name, company_list in batches.items():
    task = Task(
        description=f"批次分析：{field_name}（{len(company_list)}家）",
        prompt=f"""
        请为以下{field_name}领域的公司生成分析报告，保存到 docs/化工/{field_name}/[公司名].md：
        
        公司列表：{company_list}
        
        要求：
        1. 使用 templates/company-template.md 模板
        2. 通过Web搜索收集：官网、年报、研报、财务数据
        3. 重点标注：拐点逻辑、催化剂（价格、库存周期、政策）
        4. 每份报告完整7章节
        
        开始执行。
        """,
        readonly=False,
        subagent_type="generalPurpose",
        model="fast"  # ⭐ 关键：使用fast model加速
    )
    tasks.append(task)

# 5个批次并行执行
run_parallel(tasks)
```

#### 2.3 质量检查

每个公司报告生成后，自动检查：

✅ **必须包含的章节**：
- 公司简介（表格形式）

- 财务与业绩
- 行业地位与产业链定位
- 投资亮点
- 风险提示

✅ **必须有的数据**：
- 股票代码
- 最新年报财务数据
- 市占率或行业排名
- 至少2个信息来源

❌ **常见问题检查**：
- 是否有空白章节？
- 是否有占位符未填充（如"[待补充]"）？
- 是否缺少数据来源说明？
- 是否文件命名不规范？

#### 4.1 提取公司-概念关系

**提取逻辑**：

| 字段 | 提取来源 | 示例 |

| 信息来源 | 分析报告 | 行业分析报告+公司分析报告 |

#### 4.2 概念归属判定

**核心标的判定**（满足3项及以上）：
- 该业务营收占比 > 50%
- 市占率排名前3
- 拥有核心技术/专利壁垒
- 行业研报频繁提及为龙头

**相关标的判定**：
- 有该业务但非主营（占比 < 50%）
- 市占率排名靠后
- 转型中或布局中

## 并行处理配置

### 资源分配策略

| 配置项 | 默认值 | 说明 | 调整建议 |
|--------|--------|------|----------|
| 最大并行任务数 | 4 | 同时运行的Task数量 | API限制较紧时降到2-3 |
| 每批公司数 | 3-5家 | 每个Task分析的公司数 | 复杂公司减少到2-3家 |
| 任务超时 | 15分钟/公司 | 单个公司分析超时时间 | 简单公司可降到10分钟 |
| 失败重试 | 最多2次 | 失败后的重试次数 | 网络不稳定时增加到3次 |
| 批次间隔 | 10秒 | 两批任务之间的等待时间 | API有速率限制时增加 |

### 性能优化建议

**1. 根据公司复杂度调整**：
- 龙头企业（业务复杂）：2-3家/批
- 重要企业（中等复杂度）：3-4家/批
- 潜力企业（相对简单）：4-5家/批

**2. 根据网络状况调整**：
- 网络稳定：并行度4，每批5家
- 网络一般：并行度3，每批3-4家
- 网络不稳：并行度2，每批2-3家

**3. 根据API限制调整**：
- 无限制：并行度4，批次间隔5秒
- 有限制：并行度2-3，批次间隔20-30秒

## 错误处理与重试

### 常见错误类型

#### 错误1: 信息收集失败

**现象**：无法找到公司的财务数据或行业信息

**处理**：
1. 尝试替代数据源（东方财富、同花顺、Wind）
2. 降低数据完整性要求（仅提供核心指标）
3. 标注"数据不完整"并记录原因
4. 继续执行，不阻塞整体流程

#### 错误2: Web搜索超时

**现象**：搜索请求超时或无响应

**处理**：
1. 自动重试（最多2次）
2. 切换搜索关键词（更具体或更宽泛）
3. 如仍失败，记录日志，跳过该公司
4. 在最终报告中标注"分析失败"

#### 错误3: 文件保存失败

**现象**：无法写入文件或路径不存在

**处理**：
1. 检查文件路径是否存在，不存在则创建
2. 检查文件名是否包含特殊字符，自动清理
3. 重试保存
4. 如仍失败，使用临时文件名保存

#### 错误4: 并行任务崩溃

**现象**：某个Task异常终止

**处理**：
1. 记录崩溃的Task ID和公司列表
2. 其他Task继续执行（不影响）
3. 批次完成后，单独重新执行崩溃的Task
4. 如重试仍失败，人工介入

### 重试策略

**指数退避重试**：
```python
# 伪代码
def retry_with_backoff(func, max_retries=2):
    for i in range(max_retries + 1):
        try:
            return func()
        except Exception as e:
            if i == max_retries:
                log_error(e)
                return None
            wait_time = 2 ** i  # 1秒, 2秒, 4秒...
            time.sleep(wait_time)
```

### 数据准确性检查

**关键数据验证**：

| 数据项 | 验证方法 | 容错范围 |
|--------|----------|----------|
| 股票代码 | 6位数字 | 不容错 |
| 营收数据 | 数量级合理性 | ±20% |
| 市占率 | 0-100% | 不容错 |
| 毛利率 | -100% ~ +100% | 不容错 |
| 日期格式 | YYYY-MM-DD | 不容错 |

### 人工复核建议

**需要人工复核的情况**：

1. 第一梯队龙头企业（业务复杂，影响大）
2. 数据源之间存在严重冲突（差异>30%）
3. 公司近期有重大重组或变化
4. 自动分析标注为"低可信度"的报告

**复核重点**：
- 财务数据准确性
- 行业地位描述客观性
- 投资亮点是否有数据支撑
- 风险提示是否全面
