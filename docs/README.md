## 目录组织规范

### 新行业标准结构

#### 方式1：单层结构（公司数 < 30）
```
docs/[行业名称]/
├── 00_行业分析.md
├── 公司A.md
├── 公司B.md
└── ...
```

#### 方式2：细分结构（公司数 30-100）⭐ 推荐
```
docs/[行业名称]/
├── 00_行业概览.md          # 整体行业分析
├── [细分领域1]/
│   ├── 00_行业分析.md     # 可选：细分领域分析
│   ├── 公司A.md
│   └── ...
├── [细分领域2]/
└── ...
```

#### 方式3：深度细分（公司数 > 100）
```
docs/[行业名称]/
├── 00_行业概览.md
├── [大类1]/
│   ├── [细分1]/
│   └── [细分2]/
└── [大类2]/
```

### 命名规范

1. **行业报告**：
   - 顶层：`00_行业概览.md`（整体分析）
   - 细分：`00_行业分析.md`（细分领域分析）
   - 00开头确保排序在最前

2. **公司文件**：
   - 使用公司简称（如"康强电子"）
   - 避免全称和特殊字符

3. **目录命名**：
   - 使用中文，简洁明了
   - 按业务逻辑分类，不按字母排序

4. **链接路径**：
   - 同目录：`./公司.md`
   - 细分领域内：`./公司.md`
   - 跨目录：`../目录/文件.md`
