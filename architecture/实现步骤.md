### Docker 配置示例

```yaml
# backend/docker-compose.yml
version: '3.8'
services:
  postgres:
    image: postgres:15
    container_name: stock_postgres
    environment:
      POSTGRES_USER: postgres
      POSTGRES_PASSWORD: password
      POSTGRES_DB: stock
    ports:
      - "5432:5432"
    volumes:
      - ../data/postgres:/var/lib/postgresql/data  # 挂载到项目 data 目录
    restart: unless-stopped

# 不需要 volumes 块，使用本地目录挂载
```

> **数据持久化说明**：
> - 数据库文件存储在 `data/postgres/` 目录下
> - 即使容器停止/删除，数据依然保留
> - 下次启动容器时，数据自动恢复
> - 备份数据：`docker exec stock_postgres pg_dump -U postgres stock > data/backup.sql`
> - 清理旧数据：删除 `data/` 下的 `dragon_stock.db`、`concepts.json`、`stock_list.json` 等旧文件

## Task 3: 数据采集服务设计

**文件**: `backend/app/services/data_collector.py`, `backend/app/scheduler/`

### 架构原则

1. **后端自闭环**：数据采集作为后端服务的内置能力，不依赖外部脚本
2. **定时调度**：使用 APScheduler 集成在 FastAPI 中，服务启动即自动调度
3. **查询触发**：支持 lazy loading，查询时若数据不存在则触发采集

### 目录结构

```
backend/app/
├── scheduler/
│   ├── __init__.py
│   ├── jobs.py           # 定时任务定义
│   └── triggers.py       # 触发器（按日期、按代码等）
├── services/
│   ├── data_collector.py # 数据采集核心逻辑
│   └── data_service.py   # 数据查询服务
└── main.py               # 启动时初始化调度器
```

### 定时任务配置

```python
# backend/app/scheduler/jobs.py
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from apscheduler.triggers.cron import CronTrigger

scheduler = AsyncIOScheduler()

# 每个交易日 19:30 采集日线数据
scheduler.add_job(
    collect_daily_data,
    CronTrigger(day_of_week='mon-fri', hour=19, minute=30),
    id='daily_data_collector'
)

# 每个交易日 19:35 采集分时数据
scheduler.add_job(
    collect_intraday_data,
    CronTrigger(day_of_week='mon-fri', hour=19, minute=35),
    id='intraday_data_collector'
)

# 每个交易日 19:40 采集概念板块数据
scheduler.add_job(
    collect_concept_data,
    CronTrigger(day_of_week='mon-fri', hour=19, minute=40),
    id='concept_data_collector'
)

# 每周一 09:00 更新概念成分股映射
scheduler.add_job(
    update_concept_mapping,
    CronTrigger(day_of_week='mon', hour=9, minute=0),
    id='concept_mapping_updater'
)
```

### 查询触发机制

```python
# backend/app/services/data_service.py
async def get_stock_daily(code: str, date: str) -> dict:
    """获取股票日线数据，不存在则触发采集"""
    # 1. 先查数据库
    data = await db.query_stock_daily(code, date)
    if data:
        return data
    
    # 2. 数据不存在，触发采集
    await data_collector.collect_stock_daily(code, date)
    
    # 3. 再次查询返回
    data = await db.query_stock_daily(code, date)
    return data
```

### 优势

| 特性 | 说明 |
|------|------|
| 服务自包含 | 无需配置 crontab，启动即用 |
| 统一监控 | 所有采集任务日志统一在后端服务中 |
| 按需采集 | 查询时触发，避免无效采集 |
| 容错重试 | APScheduler 支持任务重试机制 |
| API 一致性 | 采集和查询使用相同的数据模型 |

## Task 5: 服务启动配置

### 服务启动时初始化

```python
# backend/app/main.py
from contextlib import asynccontextmanager
from apscheduler.schedulers.asyncio import AsyncIOScheduler
from app.scheduler.jobs import scheduler, init_jobs

@asynccontextmanager
async def lifespan(app: FastAPI):
    # 启动时：初始化调度器
    init_jobs()
    scheduler.start()
    yield
    # 关闭时：停止调度器
    scheduler.shutdown()

app = FastAPI(lifespan=lifespan)
```

## 错误处理规范

**文件**: `backend/app/core/exceptions.py`

### 异常类定义

```python
# backend/app/core/exceptions.py
from fastapi import HTTPException, status

class AppException(Exception):
    """应用异常基类"""
    def __init__(self, code: int, message: str, data: dict = None):
        self.code = code
        self.message = message
        self.data = data

# 具体异常类型
class DataNotFoundError(AppException):
    """数据未找到"""
    def __init__(self, message: str = "数据未找到"):
        super().__init__(code=404, message=message)

class DataCollectError(AppException):
    """数据采集失败"""
    def __init__(self, message: str = "数据采集失败"):
        super().__init__(code=500, message=message)

class ExternalAPIError(AppException):
    """外部API调用失败"""
    def __init__(self, source: str, message: str):
        super().__init__(code=503, message=f"{source} API调用失败: {message}")
```

### 全局异常处理器

```python
# backend/app/main.py
from fastapi import Request
from fastapi.responses import JSONResponse
from app.core.exceptions import AppException

@app.exception_handler(AppException)
async def app_exception_handler(request: Request, exc: AppException):
    """统一异常处理"""
    return JSONResponse(
        status_code=200,  # HTTP 状态码始终 200
        content={
            "code": exc.code,
            "message": exc.message,
            "data": exc.data
        }
    )

@app.exception_handler(Exception)
async def global_exception_handler(request: Request, exc: Exception):
    """未捕获异常的兜底处理"""
    logger.exception(f"未处理的异常: {exc}")
    return JSONResponse(
        status_code=200,
        content={
            "code": 500,
            "message": "服务器内部错误",
            "data": None
        }
    )
```

## 日志规范

**文件**: `backend/app/core/logger.py`

### 日志配置

```python
# backend/app/core/logger.py
import logging
import sys
from pathlib import Path

# 日志格式
LOG_FORMAT = "%(asctime)s | %(levelname)-8s | %(name)s | %(message)s"
DATE_FORMAT = "%Y-%m-%d %H:%M:%S"

# 配置根日志
def setup_logging(log_level: str = "INFO"):
    """配置日志系统"""
    
    # 控制台输出
    console_handler = logging.StreamHandler(sys.stdout)
    console_handler.setFormatter(logging.Formatter(LOG_FORMAT, DATE_FORMAT))
    
    # 文件输出（可选）
    log_dir = Path("logs")
    log_dir.mkdir(exist_ok=True)
    file_handler = logging.FileHandler(log_dir / "app.log")
    file_handler.setFormatter(logging.Formatter(LOG_FORMAT, DATE_FORMAT))
    
    # 配置根日志器
    logging.basicConfig(
        level=getattr(logging, log_level.upper()),
        handlers=[console_handler, file_handler]
    )

# 获取模块日志器
def get_logger(name: str) -> logging.Logger:
    return logging.getLogger(name)
```

### 日志使用规范

| 级别 | 使用场景 | 示例 |
|------|---------|------|
| DEBUG | 开发调试 | `logger.debug(f"查询参数: {params}")` |
| INFO | 正常业务流程 | `logger.info(f"采集完成: {count} 条数据")` |
| WARNING | 异常但可继续 | `logger.warning(f"数据缺失: {date}")` |
| ERROR | 业务失败 | `logger.error(f"API调用失败: {e}")` |
| CRITICAL | 系统级故障 | `logger.critical(f"数据库连接失败")` |
