# 数据库设计

## 表结构概览

| 表名 | 用途 | 数据源 |
|------|------|--------|
| `stock_info` | 股票基本信息 | adata |
| `stock_daily` | 股票日线行情 | adata |
| `stock_intraday` | 股票分时行情 | adata/tushare |
| `index_daily` | 指数日线 | adata |
| `index_intraday` | 指数分时 | adata |
| `concept_info_east` | 东方财富概念板块信息 | adata |
| `concept_info_ths` | 同花顺概念板块信息（预留） | adata |
| `concept_daily_east` | 东方财富概念板块日线 | adata |
| `concept_intraday_east` | 东方财富概念板块分时 | adata |
| `stock_concept_mapping_east` | 东方财富个股概念映射 | adata |
| `stock_concept_mapping_ths` | 同花顺个股概念映射（预留） | adata |
| `capital_flow` | 资金流向 | adata |
| `limit_list` | 涨跌停数据（含炸板信息） | Tushare |
| `account_info` | 账户信息 | 用户创建 |
| `position` | 持仓记录 | 用户交易 |
| `trade_record` | 交易记录 | 用户交易 |
| `account_snapshot` | 账户每日快照 | 系统生成 |
| **总计** | **17张表** | |

> **市场统计数据来源**：
> - 涨停/跌停/炸板数：从 `limit_list` 表聚合
> - 封板率：API 层计算 `limit_up_count / (limit_up_count + broken_board_count) * 100`
> - 最高连板：`SELECT MAX(limit_times) FROM limit_list WHERE trade_date = ? AND limit_type = 'U'`
> - 成交额：从 `index_daily` 表聚合

## concept_info_east - 东方财富概念板块信息

```sql
CREATE TABLE concept_info_east (
    concept_code VARCHAR(20) PRIMARY KEY,  -- 概念代码 (BK0612)
```

## limit_list - 涨跌停数据

```sql
CREATE TABLE limit_list (
    id SERIAL PRIMARY KEY,                -- 自增主键
    trade_date DATE NOT NULL,             -- 交易日期 (2026-03-28)
    stock_code VARCHAR(10) NOT NULL,      -- 股票代码 (002192)
    stock_name VARCHAR(20),               -- 股票名称 (融捷股份)
    limit_type CHAR(1) NOT NULL,          -- 涨跌停类型 (U=涨停, D=跌停)
    close_price DECIMAL(10,2),            -- 收盘价
    change_pct DECIMAL(8,4),              -- 涨跌幅 (%)
    first_time TIME,                      -- 首次封板时间 (09:30:00)
    last_time TIME,                       -- 最后封板时间 (14:50:00)
    open_times INTEGER DEFAULT 0,         -- 炸板次数
    limit_times INTEGER DEFAULT 0,        -- 连板高度 (1=首板, 2=2板, 3=3板...)
    limit_amount DECIMAL(20,2),           -- 封单金额 (元)
    is_broken BOOLEAN DEFAULT FALSE,      -- 是否炸过板（涨停后打开过）
    broken_time TIME,                     -- 首次炸板时间
    reseal_time TIME,                     -- 回封时间（如有）
    UNIQUE(trade_date, stock_code, limit_type)
);
CREATE INDEX idx_limit_list_date ON limit_list(trade_date);
CREATE INDEX idx_limit_list_type ON limit_list(limit_type);
CREATE INDEX idx_limit_list_times ON limit_list(limit_times);
```

> **炸板判断逻辑**：`open_times > 0` 表示炸过板。`is_broken` 为 TRUE 且 `reseal_time` 为 NULL 表示炸板未回封。
